== Introduction

Book of Praise is an application for displaying psalms and hymns as projected slides during a worship service.
It shows sheet music one verse at a time, framed in a 16:9 slide format suitable for a projector or large screen.

The application has two display modes:

SVG mode:: Shows music generated from LilyPond source files. You can edit the music and lyrics directly in the app.
PNG mode:: Shows scanned photographs of sheet music. View only.

== Installation

=== Windows

. Download the latest `bop-windows.zip` from the releases page.
. Extract the zip to a folder of your choice.
. Run `bop.exe` from the extracted folder.

Everything needed to run the application is included in the zip.

=== Linux

. Download the latest `bop-linux.zip` from the releases page.
. Extract the zip.
. Make sure GTK4 is installed on your system.
. Run the `bop` program from the extracted folder.

== Starting the Application

Double-click `bop.exe` (Windows) or run `./bop` (Linux) to start.

You can also start it from the command line with options to pre-load specific songs:

[cols="1,3"]
|===
| Option | What it does

| `--psalm 42`
| Opens with Psalm 42 loaded. Can be used more than once for multiple psalms.

| `--hymn 1`
| Opens with Hymn 1 loaded. Can be used more than once for multiple hymns.

| `--png`
| Start in photograph mode instead of the default music notation mode.
|===

.Example: start with Psalm 42 and Hymn 1
----
bop --psalm 42 --hymn 1
----

If you don't specify any songs, the application opens with Psalm 1.

== Navigation

Use the *Prev* and *Next* buttons (3) to move between slides.
The counter between them shows which slide you are on (e.g., "3/12").

You can also use the keyboard:

[cols="1,2"]
|===
| Key | Action

| Left arrow
| Previous slide

| Right arrow
| Next slide

| Ctrl+S
| Save and re-render (when the editor is open)
|===

== Building a Liturgy

The liturgy is the list of songs and verses for a service. It is shown in the summary bar (2).

=== Adding a Song

. Choose *Psalm* or *Hymn* and type the song number (4).
. The available verses appear as tick boxes.
. Tick the verses you want.
. Click *Add*.

=== Adding All Verses

Click *All* (4) to tick every verse and add the song in one step.

=== Clearing the Liturgy

Click *Clear liturgy* (3) to remove all songs and start over.

== SVG / PNG Mode

Use the toggle switch (3) to switch between modes.

[horizontal]
SVG:: Music generated from source files. You can edit the music and lyrics. The *Edit* and *Verify* buttons are available.
PNG:: Scanned photographs of sheet music. View only.

== Editing Music and Lyrics (SVG Mode)

In SVG mode you can correct the music notation and lyrics directly in the app.
Click *Edit* (3) to open the editor panel alongside the slide.

image::editor-annotated.png[Annotated editor panel,800]

[cols="1,4"]
|===
| Letter | Description

| A
| *Notes editor* -- The musical notation (notes, time signature, key) shared by all verses of the song.

| B
| *Lyrics editor* -- The words for the current verse. The label above (e.g., `lyrics_2.ly`) shows which verse you are editing.

| C
| *Save & Re-render* -- Saves your changes and regenerates the slide. You can also press *Ctrl+S*.

| D
| *Slide preview* -- Shows the result after rendering. A spinner appears while the slide is being generated. If something goes wrong, an error message appears here.
|===

=== Sending Your Edits

When you close the application after making changes, you will be asked whether to email your corrections to the maintainer.
This allows your fixes to be included in future updates.

== Verifying Verses (SVG Mode)

Click *Verify* (3) to mark the current verse as checked and correct.

- You can verify each verse once per session.
- After a verse has been verified twice (across any number of sessions), it shows *Verified* and no further action is needed.

This helps keep track of which verses have been proofread.

== Automatic Updates

The application checks for updates when it starts.

If a new version is available:

. A dialog asks whether to download and install the update.
. If you choose *Yes*, the new song files are downloaded and installed automatically.
. The song list refreshes with the updated content.

== Troubleshooting

Blank slide with no error::
The music file for that verse may be missing. In SVG mode, make sure the song folder contains the source files for that verse.

Error message on the slide::
The music source could not be processed. Click *Edit* to review and correct the files, then click *Save & Re-render*.

Application does not start::
On Windows, make sure all files from the zip are in the same folder as `bop.exe`. +
On Linux, make sure GTK4 is installed.

No songs appear::
Make sure the song data folders (`lilypond/` or `photos/`) are in the same location as the application.
